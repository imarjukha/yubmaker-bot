# 🤖 Yumbaker Bot — Инструкция по запуску

## Шаг 1. Создать Telegram бота

1. Открой Telegram, найди **@BotFather**
2. Напиши `/newbot`
3. Придумай имя бота (например: `Yumbaker Ops Bot`)
4. Придумай username (например: `yumbaker_ops_bot`)
5. BotFather выдаст **токен** — скопируй его, он понадобится

---

## Шаг 2. Получить API ключи

### Anthropic (Claude)
1. Зайди на https://console.anthropic.com
2. Раздел **API Keys** → Create Key
3. Скопируй ключ

### ClickUp
1. Зайди в ClickUp → Аватар → **Apps**
2. Раздел **API Token** → Generate
3. Скопируй токен

---

## Шаг 3. Настроить команду в config.py

Открой файл `config.py` и заполни `TEAM_MAP` — маппинг Telegram username → ClickUp user ID:

```python
TEAM_MAP = {
    "anna_smirnova": 99761883,      # Анна Смирнова
    "olga_yureva": 99763886,        # Ольга Юрьева
    "kataz0601": 99844168,          # Екатерина
    "maria_zelenova": 99761884,     # Мария Зеленова
    "nikita_akimushkin": 99761882,  # Никита Акимушкин
    "polina_zabotina": 99761939,    # Полина Заботина
    "tamara_panchenko": 99763894,   # Тамара Панченко
    "astemir": 99761885,            # Астемир Русланович
}
```

> **Как узнать Telegram username сотрудника?**
> Попроси каждого написать в личку боту `/start` — бот запишет их username автоматически.
> Или спроси у каждого их Telegram username (без @).

### Также в `services/scheduler.py`:
- Замени `GROUP_CHAT_ID` на ID вашего общего чата
- Замени `MANAGER_USERNAME` в `handlers/feedback_handler.py` на твой username

> **Как узнать ID группового чата?**
> Добавь бота в группу, напиши любое сообщение, потом открой:
> `https://api.telegram.org/bot<ТВОЙ_ТОКЕН>/getUpdates`
> Найди `"chat": {"id": -100XXXXXXXXX}` — это и есть ID группы.

---

## Шаг 4. Добавить бота в группу

1. Открой групповой чат с сотрудниками в Telegram
2. Добавь бота как участника (через Add Member)
3. Дай боту права **читать сообщения** (обычно по умолчанию)
4. Бот начнёт слушать все сообщения в группе

## Шаг 5. Задеплоить на Railway

### 5.1 Загрузи код на GitHub
```bash
git init
git add .
git commit -m "Initial commit"
git remote add origin https://github.com/твой-аккаунт/yumbaker-bot.git
git push -u origin main
```

### 5.2 Создай проект на Railway
1. Зайди на https://railway.app
2. **New Project** → **Deploy from GitHub repo**
3. Выбери репозиторий `yumbaker-bot`
4. Railway автоматически определит Python проект

### 5.3 Добавь переменные окружения
В Railway → твой проект → вкладка **Variables** → добавь:

| Key | Value |
|-----|-------|
| `TELEGRAM_TOKEN` | токен от BotFather |
| `ANTHROPIC_API_KEY` | ключ от Anthropic |
| `CLICKUP_API_TOKEN` | токен от ClickUp |
| `CLICKUP_LIST_ID` | `901217527946` |

### 5.4 Deploy
Railway задеплоит автоматически. Смотри логи во вкладке **Deployments**.

## Шаг 6. Проверить что всё работает

1. Напиши в групповой чат что-то вроде:
   `@nikita_akimushkin нужно починить кондиционер до пятницы`

2. Бот должен ответить: ✅ Задача создана в ClickUp

3. Если исполнитель не упомянут:
   `Нужно закупить продукты до завтра`
   Бот должен показать кнопки выбора исполнителя

## Как работает бот

### Обнаружение задач
Бот анализирует каждое сообщение через Claude AI.
Задача — это сообщение с конкретным действием: "починить", "сделать", "закупить", "договориться" и т.п.
Обычный разговор бот игнорирует.

### Назначение исполнителя
- Если в сообщении есть `@username` — назначает автоматически
- Если нет — показывает кнопки с именами команды

### Фидбэк
Бот пишет исполнителю в личку:
- Задача < 7 дней → за 1 день до дедлайна
- Задача 7–14 дней → посередине + за 1 день до дедлайна
- Задача > 14 дней → каждые 5 дней + за 1 день до дедлайна

### Еженедельный отчёт
Каждую **пятницу в 18:00**:
- Каждому сотруднику → подробный личный отчёт
- В общий чат → краткая сводка с оценками 🟢🟡🔴

## Возможные проблемы

**Бот не видит сообщения в группе**
→ Убедись что бот добавлен в группу и имеет право читать сообщения

**Бот не может написать сотруднику в личку**
→ Сотрудник должен сначала написать боту `/start` в личку

**Ошибка при создании задачи в ClickUp**
→ Проверь CLICKUP_API_TOKEN и CLICKUP_LIST_ID в переменных Railway
